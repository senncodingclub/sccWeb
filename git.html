How to use git to manage your HTML pages

Git is an awesome version control system or VCS. The point of VCS is so that you can keep track of your projects and code. You might have heard of Github before and probably think that Git and Github is the same thing. This is a very common misconception, but they are not the same thing. Git the is the protocol that is used for VCS, while Github supports that protocol and is useful for backing up your code. Github is the industry standard for open source projects and is a really great time killer if you want to explore awesome stuff that people are creating. The way that we can use Git and Github is by using the Git command line tool to upload our projects to Github (which weâ€™ve already done).

To download your project from Github:

  1. Download the Git command line from here
  2. Install Git using the default options - you don't need to change anything
  3. If you are on a Windows machine:
    • Open the start menu and search for Git
    • Open the program called Git Bash
    • Type in the command git --version and press enter
    • If the output is something like git version 1.9.4..., then you're good!
    • If not, then you should try reinstalling Git
  4. If you are on a Mac:
    • Do a spotlight search for terminal and open it
    • Type in the command git --version and press enter
    • If the output is something like git version 1.9.4..., then you're good!
    • If not, then you should try reinstalling Git
  5. With the terminal or Git Bash open, we have to change into a directory where we would like to download our project into.
  6. To change directories (directory = folder), we use the command cd
  7. We can use a multitude of directories to store our projects, but I like to use the Downloads folder, which is located in ~/Downloads.
  8. You can use another directory as well if you wish such as:
    • Desktop: ~/Desktop
    • Downloads: ~/Downloads
    • And so on and so forth....
  9. To change into a directory, use the command cd [Name of directory]
    • To change into the Downloads folder, type in the command cd ~/Downloads Make sure you pay attention to the capilization when entering these commands!
  10. Now to actually download the code we have uploaded into the Github repository, we run the command git clone https://github.com/[github username]/[name of the repository]
    • When we created the Github repo at coding club, we all named it senncodingclub, but login to your github to make sure it isn't different in terms of capitalization or the name itself.
    • To download our code, we can run the command git clone https://github.com/[github username]/[name of the repository] and press enter.
    • E.G., for my repo, I can run the command git clone https://github.com/rakeshdas1/senncodingclub, since my Github username is rakeshdas1 and my repo's name is senncodingclub
  11. Now the code should be downloaded into a folder named whatever you named your repo

To edit your HTML and make an awesome webpage:

  1. Download Brackets from here.
  2. Install Brackets with all the default options.
  3. Open Brackets
  4. Change the working directory by clicking on the Getting Started link on the upper left hand corner and choosing where you downloaded your code.
  5. Click on your index.html file on the upper right hand corner.
  6. Edit your HTML page and use W3Schools as a reference point.
  7. Save all your work

To push your changes backup to Github:

  1. Open up Git Bash or your terminal again if you closed it
  2. cdinto the directory where you downloaded your code
    • E.G., I downloaded my code into my downloads folder, so I can run the command cd ~/Downloads/senncodingclub
  3. We have to inform Git of which files we changed by running the command git add index.html.
  4. Now we have to commit a change, which is basically a state of your project at a certain time. With this commit, we also have to attach a commit message, which is a small description of what you changed.
    • To commit, we can run the command git commit -m "your commit message".
    • E.G., I ran the command git commit -m "Added an image" with my project.
  5. Run the command git push to push your code back up to Github
    • This command will ask for your username and password to Github, so just enter those in when you are prompted to do so.
  6. You're done! You can check your Github repo on Github's website now.
